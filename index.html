Forma de trabalho do git

Working Directory: É o diretório aonde você vai estar trabalhando, esse diretório é criado quando você vai lá no repositório e pega a base do sistema traz ele para sua máquina pra começar a ajudar o desenvolvimento, então tudo que você estiver fazendo, tudo que você estiver codificando naquela hora está dentro do primeiro estágio.

Staging Area (git add): Arquivos que estavam sendo editados passam para uma sala de espera que é o staging area, essa sala de espera é onde os arquivos que você já modificou ficam antes de serem enviados de volta para o repositório quando você der git commit esses arquivos vão ir para o repositório final.

Repository (git commit): Esse é o estágio final que é o repositório principal do projeto, e é justamente essa divisão, esses estágios que nós acabamos de ver que dão alguém de uma flexibilidade uma confiabilidade muito maior em relação aos outros sistemas de controle de versão.
